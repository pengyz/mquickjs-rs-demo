# 规划/决策记录（planning）

本目录用于存放**按工作推进维度**记录的设计讨论、方案比较、决策过程与阶段性结论。

## 重要说明

- 本目录内容 **不是** 现行规范（Source of Truth）。
- 现行口径以 `docs/` 下对应功能文档为准（尤其是 `docs/ridl/`、`docs/build/`、`docs/architecture/`）。
- 每篇 planning 文档建议在开头包含：
  - 状态：已落地 / 部分落地 / 已废弃
  - 现行替代：指向对应的功能文档
  - 关键结论（3~5 条）
