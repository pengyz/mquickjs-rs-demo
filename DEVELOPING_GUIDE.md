# 开发指南（DEVELOPING_GUIDE）

本项目是 QuickJS + Rust 的 RIDL 示例。开始开发前请先阅读本文件。

## 前置要求
- Rust 工具链（edition 2021），建议用 rustup 安装。
- Cargo 可运行构建脚本（`build.rs`），一般无需额外系统依赖。
- 编辑器启用 rust-analyzer；本地可运行 `cargo fmt` / `cargo check`。

## 规范和约束
- 对于我给出的需求，永远要先尽可能深度思考，给出详细计划，我们的计划放在根目录的doc/planning目录中，每个需求一个计划，计划做完后要在文档中标记当前plan已完成。
- 你给出工作计划后，我们会针对该计划进行讨论，在我明确告知你需求已就绪时，才能开始开发。
- 开发过程中，遇到问题，优先询问我，不要盲目猜测
- 任何的需求开发都需要编写测试用例，你需优先完成此项工作，且明确通知我审阅测试用例，需求开发要求所有测试用例都能通过，如果遇到测试用例无法通过，请先向我汇报原因，由我决策后续如何修复
- 每次你完成功能开发，都需要更新相关文档，确保文档和代码是一致的，如果不一致，先向我汇报问题，由我确认后再进行修正
- 为每个大的模块创建一个README.md文档，描述该模块的作用。如果模块较复杂，还可给出设计文档，实施文档等
- 每次新建上下文，都需要优先读取此文档，按照其中的知识开展工作
- 此文档是我手动编辑的，你绝对不修改它，更不能删除该文档
